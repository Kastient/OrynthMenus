# OrynthMenus

OrynthMenus — это **гибкий конструктор инвентарных меню и Bedrock-форм** для серверов на `NukkitMOT123`.

Плагин позволяет создавать полноценные серверные интерфейсы через обычные YAML-файлы — **без изменения Java-кода**, повторной сборки проекта и ручного программирования каждого меню.

> 💡 **Главная идея:** вы описываете внешний вид, условия и действия в YAML, а OrynthMenus берёт на себя отображение интерфейса, проверки и безопасное выполнение операций.

### ✨ Коротко о возможностях

| Возможность         | Что даёт                                                                |
| ------------------- | ----------------------------------------------------------------------- |
| 🧰 Инвентарные меню | Магазины, скупщики, навигация, донат-витрины и другие интерфейсы        |
| 📱 Bedrock-формы    | Информационные окна, подтверждения, выбор режима и списки привилегий    |
| ⚙️ Условия          | Проверка прав, денег, предметов, опыта и пользовательских требований    |
| 🎯 Действия         | Команды, сообщения, звуки, выдача наград, открытие и обновление меню    |
| 🔄 Динамика         | Автоматическое обновление баланса, опыта и плейсхолдеров                |
| 🔌 Java API         | Подключение собственных требований и управление меню из других плагинов |

### 🚀 Что можно создать

С помощью OrynthMenus можно собрать:

* **главное меню сервера**;
* **магазин предметов**;
* **скупщика ресурсов**;
* **донат-магазин**;
* меню телепортации;
* выбор режима;
* список привилегий;
* ежедневные награды;
* информационные разделы;
* интерфейсы сторонних плагинов.

> ✅ Один плагин подходит как для простых информационных окон, так и для сложных магазинов с проверками, приоритетами и несколькими цепочками действий.

### 🧰 Инвентарные меню

Создавайте классические меню Minecraft на основе различных контейнеров:

* сундуки на `27` и `54` слота;
* воронки;
* печи;
* бочки;
* шалкеры;
* маяки;
* эндер-сундуки;
* другие контейнеры, поддерживаемые FakeInventories.

Для каждого меню можно настроить:

* собственный заголовок;
* размер и тип контейнера;
* команду открытия;
* интервал обновления;
* расположение элементов;
* условия отображения;
* действия при нажатии.

Игрок увидит заданное вами название меню, а не стандартное название контейнера вроде `Chest`.

### 📱 Bedrock-формы

OrynthMenus поддерживает Bedrock `SimpleForm` с:

* заголовком;
* многострочным описанием;
* кнопками;
* подписями кнопок;
* обработкой нажатий;
* открытием других меню;
* выполнением команд и действий.

**Лучше всего формы подходят для:**

1. информации о донате;
2. выбора режима;
3. подтверждения действия;
4. навигации;
5. вывода правил и подсказок.

> ⚠️ **Ограничение Bedrock:** общий текст формы всегда отображается выше списка кнопок. Вставить настоящую кнопку между двумя абзацами нельзя — это ограничение клиента.

### ⚙️ Гибкая логика

Для каждого элемента можно отдельно настроить:

* `view_requirement` — будет ли предмет показан;
* `click_requirement` — можно ли его нажать;
* `deny_action` — что произойдёт при отказе;
* `success_action` — что произойдёт после успешной проверки;
* `action` — основная цепочка действий.

Поддерживаются проверки:

* наличия права;
* отсутствия права;
* баланса игрока;
* наличия предмета;
* уровня опыта;
* пользовательских условий от сторонних плагинов.

#### Пример логики покупки

```yaml
click_requirement:
  requirements:
    money:
      type: "has money"
      amount: 5000

  deny_action:
    - "[message] &cНедостаточно монет."
    - "[sound] NOTE_BASS"

success_action:
  - "[sound] RANDOM_LEVELUP"

action:
  - type: take_money
    amount: 5000
  - type: give_item
    material: "minecraft:diamond"
    amount: 1
  - "[message] &aВы купили алмаз."
```

### 🎯 Система действий

Действия выполняются **последовательно, сверху вниз**.

Поддерживаются:

* `[console]` — команда от имени консоли;
* `[player]` — команда от имени игрока;
* `[message]` — сообщение в чат;
* `[title]` — заголовок на экране;
* `[actionbar]` — текст над хотбаром;
* `[sound]` — воспроизведение звука;
* `[open]` — открытие другого меню;
* `[close]` — закрытие меню;
* `[refresh]` — обновление текущего меню.

Также доступны безопасные YAML-действия:

```
give_item
take_item
give_money
take_money
give_experience
take_experience_levels
effect
give_permission
remove_permission
```

> 🛡️ Если безопасное действие выполнить не удалось, дальнейшая цепочка автоматически останавливается. Это защищает игроков от неправильных списаний и дублирования наград.

### 🔄 Динамические элементы

Отдельные предметы могут обновляться прямо в открытом меню.

Это удобно для отображения:

* текущего баланса;
* недостающих монет;
* уровня опыта;
* данных из PlaceholderAPI;
* статуса покупки;
* доступности привилегии.

Чтобы включить обновление элемента, добавьте:

```yaml
update: true
```

Интервал задаётся на уровне меню:

```yaml
update_interval: 1
```

> ⚡ Обновляются только элементы с `update: true` и только пока меню действительно открыто у игрока.

### 💎 Приоритет элементов

В одном слоте можно описать несколько вариантов предмета.

Параметр `priority` определяет, какой вариант будет показан: **чем больше число, тем выше приоритет**.

Это удобно для меняющихся кнопок:

* `Купить`;
* `Уже приобретено`;
* `Недостаточно монет`;
* `Доступно только VIP`.

```yaml
priority: 10
```

### 📦 Зависимости

| Плагин                |          Статус | Назначение                              |
| --------------------- | --------------: | --------------------------------------- |
| FakeInventories       | **Обязательно** | Создание и отображение инвентарных меню |
| EconomyAPI            |   Для экономики | Проверка, выдача и списание денег       |
| PlaceholderAPI-nukkit |   Необязательно | Поддержка внешних плейсхолдеров         |

> ℹ️ Без EconomyAPI обычные меню продолжат работать, но денежные проверки и операции будут недоступны.

### 🛠 Быстрый пример

Простейшее меню, которое показывает баланс игрока:

```yaml
type: inventory
inventory-type: CHEST
size: 27
title: "&8Магазин"
update_interval: 1

open:
  command: shop
  description: "Открыть магазин"

items:
  balance:
    material: "minecraft:emerald"
    slot: 15
    display_name: "&aВаш баланс"
    lore:
      - "&fМонеты: &e%OrynthMenus_balance%"
    update: true
```

После регистрации меню игрок сможет открыть его командой:

```
/shop
```
