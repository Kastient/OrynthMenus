# Команды и permissions

[← На главную](../README.md) · [Установка](INSTALLATION.md) · [Конфигурация →](CONFIGURATION.md)

## Основная команда

Полное имя — `/orynthmenus`, алиас — `/omenus`.

| Команда | Кто может выполнить | Permission | Описание |
| :--- | :--- | :--- | :--- |
| `/omenus help` | Игрок или консоль | — | Показывает справку |
| `/omenus settings` | Только игрок | `orynthmenus.admin.settings` | Открывает административную Bedrock-панель |
| `/omenus open <игрок> <menu-id>` | Игрок или консоль | `orynthmenus.admin.open` | Принудительно открывает меню выбранному игроку |
| `/omenus validate` | Игрок или консоль | `orynthmenus.admin.validate` | Проверяет все зарегистрированные YAML без применения |
| `/omenus reload` | Игрок или консоль | `orynthmenus.admin.reload` | Безопасно перезагружает конфигурацию |

Права из `plugin.yml` имеют `default: op`.

## Команды отдельных меню

Любое меню может зарегистрировать свою команду:

```yaml
open:
  command: shop
  permission: "server.menu.shop"
  description: "Открыть магазин"
  action:
    - "[sound] random.chestopen"
```

После reload появится `/shop`. Permission из `open.permission` проверяется:

- при открытии команды меню;
- при переходе через `[open]`/`open_menu`;
- при вызове обычного `OrynthMenusApi#open`.

Административная команда `/omenus open` намеренно обходит `open.permission`, так как предназначена для принудительного открытия.

Имена динамических команд допускают только латинские буквы нижнего регистра, цифры, `_` и `-`. Конфликт с существующей командой выводится в консоль; чужая команда не перезаписывается.

## Bedrock-подсказки

Плагин публикует клиенту overload-подсказки для `help`, `settings`, `reload`, `validate` и `open`. Для аргумента `menu` передаются ID загруженных меню, а для игрока используется тип `TARGET`.

## Встроенная панель `/omenus settings`

Главный экран содержит четыре раздела:

1. настройки плагина;
2. список меню;
3. безопасная перезагрузка;
4. подсказки по командам.

### Настройки плагина

Из формы можно изменить:

- язык `RU`/`EN`;
- экономический провайдер `EconomyAPI`, `OrynthEconomy`, `AUTO`, `NONE`;
- валюту OrynthEconomy: `regular`/`donate`.

### Редактор Inventory GUI

В списке меню выберите инвентарное меню и пункт «Редактировать дизайн»:

1. нажмите предмет-кнопку;
2. нажмите новый слот;
3. если слот занят, кнопки поменяются местами;
4. изменение сразу записывается в YAML.

Пустые слоты редактора визуально заполняются стеклом. Перед первой записью создаётся резервная копия. Редактор меняет только расположение кнопок, не затрагивая действия и условия.

### Редактор Bedrock Form

Для формы можно изменить:

- заголовок;
- многострочное содержимое;
- текст кнопки;
- две строки подписи;
- порядок кнопок.

## Права, выдаваемые действиями

`give_permission` не требует внешнего permission-плагина. Постоянные права записываются в `plugins/OrynthMenus/permissions.yml` по UUID и восстанавливаются при входе игрока.

```yaml
- type: give_permission
  permission: "server.feature.premium"
```

Временный вариант существует только до отключения/выхода игрока:

```yaml
- type: give_permission
  permission: "server.feature.preview"
  temporary: true
```

Для удаления используйте `remove_permission` с тем же полем `temporary`, которое использовалось при выдаче.
