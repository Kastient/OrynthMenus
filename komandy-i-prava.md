# Команды и права

## 🧾 Команды и права

OrynthMenus автоматически регистрирует команды открытия меню и содержит отдельную административную команду для управления конфигурацией.

### Основные команды

| Команда               | Назначение                            |
| --------------------- | ------------------------------------- |
| `/menu`               | Открыть главное меню                  |
| `/buyer`              | Открыть меню скупщика                 |
| `/donshop`            | Открыть донат-магазин                 |
| `/donate`             | Открыть форму со списком привилегий   |
| `/orynthfeatures`     | Открыть демонстрационное меню         |
| `/orynthmenus reload` | Перезагрузить конфигурацию            |
| `/omenus reload`      | Короткий вариант команды перезагрузки |

> ℹ️ Команды `/menu`, `/buyer`, `/donshop` и другие зависят от установленных конфигов. Вы можете создавать собственные команды для любых меню.

### Команда перезагрузки

Полная команда:

```
/orynthmenus reload
```

Короткий вариант:

```
/omenus reload
```

Перезагрузка обновляет:

* список зарегистрированных меню;
* содержимое YAML-файлов;
* действия;
* требования;
* названия;
* команды открытия;
* системные настройки.

> ⚠️ После обновления самого `.jar` всегда выполняйте полный перезапуск сервера, а не только `reload`.

### Право администратора

Для перезагрузки требуется право:

```
orynthmenus.admin.reload
```

Пример выдачи через LuckPerms:

```
/lp user Player permission set orynthmenus.admin.reload true
```

Для группы администраторов:

```
/lp group admin permission set orynthmenus.admin.reload true
```

### Создание собственной команды

Команда задаётся внутри файла меню:

```yaml
open:
  command: shop
  description: "Открыть магазин"
```

После перезагрузки появится команда:

```
/shop
```

### Команда с другим названием

```yaml
open:
  command: servermenu
  description: "Открыть главное меню сервера"
```

Игрок будет использовать:

```
/servermenu
```

### Команды внутри действий

OrynthMenus может выполнять команды от имени консоли:

```yaml
action:
  - "[console] give %player_name% diamond 1"
```

Или от имени игрока:

```yaml
action:
  - "[player] spawn"
```

#### Когда использовать `[console]`

Используйте консольные команды для:

* выдачи предметов;
* изменения групп;
* выдачи валюты;
* телепортации через админ-команды;
* выполнения действий без прав у игрока.

#### Когда использовать `[player]`

Используйте команды игрока, если:

* команда должна учитывать права игрока;
* команда должна выполняться именно от его имени;
* сторонний плагин обрабатывает контекст игрока.

### Плейсхолдер имени игрока

В командах можно использовать:

```
%player_name%
```

Пример:

```yaml
action:
  - "[console] give %player_name% emerald 16"
  - "[message] &aВы получили 16 изумрудов."
```

### Защита команд

Рекомендуется:

* не давать игрокам право на административный `reload`;
* не использовать команды с неконтролируемым вводом;
* проверять требования перед выдачей наград;
* использовать безопасные действия вместо команд, когда это возможно.

> 🛡️ Для покупок лучше применять `take_money` и `give_item`, а не собирать всю логику только на консольных командах.

### Быстрая проверка

* [ ] Команда открытия уникальна.
* [ ] После изменения выполнен `/orynthmenus reload`.
* [ ] Администратору выдано нужное право.
* [ ] В командах используется `%player_name%`.
* [ ] Для покупок настроены требования.
* [ ] Административные действия не доступны обычным игрокам.
